# Implementation Notes: 023

**Table format** (plain ASCII, no Unicode box chars):
```
+----+--------------+-------+-----------+----------+-----------------+
| #  | Session ID   | Depth | Status    | Duration | Team            |
+----+--------------+-------+-----------+----------+-----------------+
|  1 | sess-abc123  |     1 | completed |    12.5s | workers         |
|  2 | sess-def456  |     2 | failed    |     3.1s | -               |
+----+--------------+-------+-----------+----------+-----------------+
```
Use ASCII `-` rather than em dash for the "no team" indicator to avoid encoding issues.

**Column widths** (minimum, expand if content is wider):
- `#`: 4
- `Session ID`: 12
- `Depth`: 5
- `Status`: 9 (longest value: `completed`)
- `Duration`: 8
- `Team`: 15

**stderr printing**: `print(..., file=sys.stderr)`. Add `import sys` if not already present.

**Placement**: Define `_print_status_table()` immediately after `_log_entry()`.

**Error isolation**: Wrap the entire function body in:
```python
try:
    ...
except Exception:
    pass
```
Status table printing is best-effort and must not crash the MCP server.

**Call placement in call_tool()**: Insert just before the final `return [TextContent(...)]` statement. Must execute even in the timeout path — add the call in the timeout exception handler too (after the session registry entry is appended there).
